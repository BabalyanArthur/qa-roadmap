Конспект для заучивания: Основы тестирования

Что такое тестирование ПО

Процесс анализа программы с целью найти дефекты и повысить качество продукта. Тестирование показывает, соответствует ли реальное поведение системы ожидаемому.

Кто такой тестировщик

Специалист, который ищет различия между тем, какой система должна быть, и тем, какая она есть на самом деле. Предоставляет информацию о качестве продукта, чтобы команда могла принимать решения.

Что такое баг (дефект)

Отклонение фактического результата от ожидаемого. Если в требованиях написано «кнопка красная», а она синяя — это баг.

Что такое тест-кейс

Пошаговая инструкция для проверки одной конкретной функции. Содержит: предусловия, шаги, ожидаемый результат.

Что такое чек-лист

Список проверок без подробных шагов. Гибче тест-кейса, используется когда нет времени писать подробные сценарии.

Виды тестирования

Функциональное: проверяет, что система делает то, что заявлено (логин, поиск, оплата).
Нефункциональное: проверяет, как система работает (скорость, удобство, надёжность).
Регрессионное: проверяет, не сломали ли старые функции после добавления новых.
Smoke-тестирование: быстрая проверка критических функций перед основным тестированием.
Клиент-серверная архитектура

Клиент (браузер, мобильное приложение) отправляет запрос, сервер его обрабатывает и возвращает ответ. API — это интерфейс, через который клиент и сервер общаются.

HTTP-методы

GET — получить данные от сервера.
POST — отправить новые данные на сервер.
PUT — обновить существующие данные.
DELETE — удалить данные.
JSON

Формат обмена данными. Выглядит как текст в фигурных скобках: {"name": "doggie", "status": "available"}.

Git

Система контроля версий. Сохраняет историю изменений кода, позволяет откатываться назад и работать в команде без конфликтов.

Терминал

Текстовый интерфейс для управления компьютером через команды.
